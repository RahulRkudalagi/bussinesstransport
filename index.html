Key Features

Time Lookup Between Places

Search for travel times between two locations using pre-calculated or real-time data.

Exploring Nearby Locations (DFS)

Use Depth First Search (DFS) to suggest nearby places for casual exploration.

Shortest Path Finder (Dijkstra’s Algorithm)

Find the optimal path between two locations using Dijkstra’s algorithm for minimum travel time.

Level-by-Level Travel (BFS)

Explore locations level by level using Breadth First Search (BFS) to gradually expand your route.

Connected Shortest Path Tour (Prim’s Algorithm)

Visit all places with the shortest connected paths using Prim’s algorithm.

Disconnected Shortest Path Tour (Kruskal’s Algorithm)

Cover all locations in the shortest possible paths, even if they’re not connected, using Kruskal’s algorithm.

Algorithms Used

Various algorithms have been implemented to enhance the efficiency of the system:

  • Dijkstra’s Algorithm: Optimal pathfinding for shortest routes.
  • DFS (Depth First Search): Explore nearby locations without worrying about the path.
  • BFS (Breadth First Search): Level-by-level exploration of places.
  • Prim’s Algorithm: Ensure a connected shortest path tour.
  • Kruskal’s Algorithm: Find the shortest paths even when a connected tour isn’t needed.
